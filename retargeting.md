# 핸드 리타게팅 개선 개발 배경 및 구조

**작성일:** 2026.03.27  
**목적:** 리타게팅 고도화 개발의 필요성, 방법론 선정 근거, 개발 구조 설명

---

## 1. 왜 리타게팅을 개선해야 하는가

### 1.1 문제의 출발점

현재 원격조작 시스템에서 작업자의 손 동작을 로봇 핸드(Tesollo DG-5F)에 전달하는 리타게팅은 시스템의 **파지 성능 병목**이다. 리타게팅 품질이 낮으면 아무리 좋은 원격조작 인터페이스와 로봇 하드웨어가 있어도, 작업자의 의도가 로봇에 정확히 전달되지 않아 파지 실패, 물체 낙하, 비효율적 재시도가 반복된다.

### 1.2 리타게팅 품질이 영향을 미치는 범위

```
리타게팅 품질 ──→ 파지 성공률
              ──→ 원격조작 작업 시간
              ──→ 작업자 인지 부하
              ──→ 시연 데이터 품질 ──→ 모방학습 정책 성능
              ──→ Shared Autonomy 보조 효과 (Residual Copilot 등)
```

이전 문헌 조사에서 확인된 바와 같이, Residual Copilot(Sha 2026)은 "copilot 보조 하에 수집된 시연이 다운스트림 모방학습 성능을 대폭 향상시킨다"는 것을 보였다. 리타게팅도 동일한 원리로, **리타게팅 품질 향상 → 시연 품질 향상 → 자율 정책 성능 향상**의 연쇄 효과를 가진다.

### 1.3 현재 상태와 개선 여지

리타게팅 방법론은 5세대에 걸쳐 발전해왔으며(별첨: 핸드 리타게팅 방법론 발전역사), 현재 `manusGlove` ROS2 패키지의 리타게팅 세대 및 성능이 미확인 상태이다. 최신 3세대(다중 제약 최적화) 방식은 2세대(Fingertip IK) 대비 파지 형태 보존, 자기충돌 방지, 물리적 실현 가능성에서 유의미한 개선을 보이며, 이를 적용하면 별도 하드웨어 추가 없이 SW 개선만으로 파지 성능을 향상시킬 수 있다.

## 2. 왜 이 방법론들이 선정되었는가

### 2.1 선정된 방법론 요약

| 순서 | 방법론 | 세대 | 역할 |
|---|---|---|---|
| ① | Direct Mapping (`models/direct_mapping`) | 1세대 | 성능 비교 기준선(baseline) |
| ② | AnyTeleop (Qin et al., RSS 2023) | 3세대 | 메인 리타게팅 시스템 |
| ③ | Retargeting Objective 분석 (Yu et al., 2025) | 3세대 고도화 | 비용 함수 최적화 |
| (향후) | TypeTele (Lin et al., CoRL 2025) | 5세대 | 추가 성능 고도화 검증 |

### 2.2 AnyTeleop을 메인으로 선정한 이유

**후보 비교:**

| 후보 | 장점 | 단점 | 선정 여부 |
|---|---|---|---|
| DexPilot (ICRA 2020) | 3세대의 출발점, 검증된 성능 | Allegro Hand 특화, 컬러 장갑 필요 | ✗ |
| **AnyTeleop (RSS 2023)** | **다양한 핸드 지원, 오픈소스, 표준 프레임워크** | 특정 과제 최적화 없음 | **✓** |
| TypeTele (CoRL 2025) | 5세대, 리타게팅 한계 돌파 | 타입 라이브러리 구축 비용, 초기 검증 단계 | 향후 |

**AnyTeleop 선정 근거:**

1. **핸드 범용성:** Allegro, LEAP, xArm 등 다양한 핸드에서 검증됨. DG-5F 적용 시 키포인트만 재정의하면 됨.
2. **오픈소스:** 코드가 공개되어 있어 구현 시간을 단축할 수 있음. 카메라 입력부만 Manus 입력으로 교체하면 핵심 최적화 엔진을 재사용 가능.
3. **사실상 표준:** 2023년 이후 대부분의 최신 원격조작 논문(DexCap, Open-TeleVision, Bunny-VisionPro 등)이 AnyTeleop의 리타게팅을 사용하거나 참조. 검증 사례가 가장 풍부.
4. **점진적 확장 가능:** AnyTeleop의 비용 함수 위에 Retargeting Objective 분석의 최적화를 추가하고, 이후 5세대 잔차 보정까지 확장하는 경로가 자연스러움.

### 2.3 Retargeting Objective 분석을 고도화에 선정한 이유

AnyTeleop은 범용 프레임워크이므로 DG-5F + 특정 과제에 최적화되어 있지 않다. Yu et al. (2025)은 비용 함수의 각 항(fingertip 위치, 손가락 간 거리, 손가락 방향, 자기충돌 등)이 **실제 조작 성능에 미치는 영향을 체계적으로 분석**한 유일한 연구이다. 이 분석 방법론을 DG-5F에 적용하면, "어떤 비용 항이 키 파지에 가장 중요한지"를 실험적으로 밝힐 수 있다.

### 2.4 1세대(Direct Mapping)을 기준선으로 사용하는 이유

3세대의 개선 효과를 **정량적으로 증명**하려면 비교 기준이 필요하다. 1세대 Direct Mapping (`models/direct_mapping`)은 구현이 가장 간단하고 IK/최적화 없이 각도 매핑만으로 동작하여, "최적화 없이 가장 단순한 방식만 쓰면 어떤 수준인지"를 보여주는 명확한 기준선이 된다. 이를 통해 "3세대 도입으로 파지 성공률이 X% → Y%로 향상되었다"는 정량 근거를 확보할 수 있다. (별도의 2세대 Fingertip IK 구현체는 개발하지 않는다 — 같은 fingertip 정보가 3세대 multi-cost 최적화에 cost 항으로 통합되므로.)

### 2.5 5세대(TypeTele)를 당장 개발하지 않는 이유

TypeTele은 DG-5F 전용 **조작 타입 라이브러리**를 새로 구축해야 하며, 이 라이브러리의 품질이 성능을 좌우한다. 3세대 리타게팅이 아직 구현/검증되지 않은 상태에서 5세대로 넘어가면, 기준선 없이 효과를 판단할 수 없다. 3세대를 먼저 확보한 후, **3세대로 부족한 과제가 식별될 때** 5세대를 도입하는 것이 효율적이다.

## 3. 무엇을 개발하는가 — 개발 항목 구조

### 3.1 전체 구조

```
[최종 목표] 리타게팅 품질 향상으로 원격조작 파지 성공률 개선

├── [대분류 1] 입력 변환 시스템
│   └── 모든 리타게팅 방법의 공통 기반
│
├── [대분류 2] 리타게팅 알고리즘
│   ├── 1세대 기준선
│   ├── 3세대 메인 시스템
│   └── 3세대 성능 고도화
│
└── [대분류 3] 성능 평가 체계
    └── 정량적 비교 및 개선 효과 입증
```

### 3.2 대분류별 상세

---

#### 대분류 1: 입력 변환 시스템

**왜 필요한가:** AnyTeleop 등 리타게팅 알고리즘은 인간 손의 3D 키포인트를 입력으로 요구하지만, Manus Glove는 자체 포맷(skeleton 21 joints)으로 데이터를 제공한다. 이 간극을 연결하는 변환기가 모든 리타게팅 방법의 **공통 전제 조건**이다.

| 소분류 | 개발 항목 | 목적 |
|---|---|---|
| 1-1 | Manus SDK 출력 포맷 분석서 | Manus 데이터의 관절 순서, 좌표계, 단위를 명확히 파악 |
| 1-2 | Manus-MANO 관절 매핑 테이블 | Manus 관절 인덱스와 리타게팅 표준(MANO) 인덱스의 1:1 대응 확립 |
| 1-3 | 좌표계 변환 행렬 | Manus 좌표계 → 리타게팅 표준 좌표계 변환 정의 |
| 1-4 | `manus_keypoint_converter` ROS2 노드 | 위 1-1~1-3을 통합한 실시간 변환 노드 |

**의존 관계:** 대분류 2, 3의 모든 항목이 이 변환기에 의존.

---

#### 대분류 2: 리타게팅 알고리즘

**왜 필요한가:** 변환된 키포인트를 DG-5F의 관절 명령으로 변환하는 핵심 알고리즘. 세대별로 구현하여 점진적 개선을 검증한다.

**소분류 2-A: 1세대 Direct Mapping 기준선** (`models/direct_mapping`)

| 소분류 | 개발 항목 | 목적 |
|---|---|---|
| 2-A-1 | DG-5F URDF/joint metadata 정비 | 리타게팅 대상 로봇 핸드의 관절 한계/이름 확보 (`models/direct_mapping/dg5f_fk.py`) |
| 2-A-2 | Direct Mapping 모델 + 캘리브레이션 | 각도 추출 → baseline subtraction → scale → joint clip. 3세대와의 비교 기준선 역할 |

→ **이것을 개발하는 이유:** 3세대 개선 효과의 정량적 입증을 위한 비교 기준 확보. (별도 2세대 Fingertip IK 솔버는 만들지 않음 — 같은 정보가 3세대 cost 항으로 통합되므로.)

**소분류 2-B: 3세대 AnyTeleop (메인 시스템)**

| 소분류 | 개발 항목 | 목적 |
|---|---|---|
| 2-B-1 | DG-5F 전용 키포인트 정의서 | DG-5F의 fingertip, 관절, 손바닥 키포인트를 리타게팅 비용 함수에 사용할 수 있도록 정의 |
| 2-B-2 | AnyTeleop 비용 함수 구현 | fingertip 위치 + 손가락 쌍 거리 + 자기충돌 + 관절 범위 제약의 다중 제약 최적화 |
| 2-B-3 | 실시간 최적화 솔버 통합 | <10ms/프레임 달성을 위한 솔버 선택 및 warm-start 구현 |
| 2-B-4 | `retarget_anyteleop` ROS2 노드 | 위 2-B-1~3을 통합한 3세대 리타게팅 노드 |

→ **이것을 개발하는 이유:** 현재 주류(mainstream)인 3세대 리타게팅을 확보하여, 파지 형태 보존·자기충돌 방지·물리적 실현 가능성을 구조적으로 보장.

**소분류 2-C: 3세대 비용 함수 최적화 (Retargeting Objective)**

| 소분류 | 개발 항목 | 목적 |
|---|---|---|
| 2-C-1 | 비용 항 ablation 실험 | 각 비용 항의 DG-5F에서의 실제 기여도 측정 |
| 2-C-2 | DG-5F + 과제 최적 가중치 도출 | ablation 결과 기반 최적 가중치 결정 |
| 2-C-3 | 과제별 가중치 프리셋 | power grasp, precision pinch, lateral pinch 등 과제별 최적 설정 |

→ **이것을 개발하는 이유:** 범용 AnyTeleop을 DG-5F와 우리 과제(키 파지, 밸브 파지 등)에 **특화**하여 추가 성능 향상. "어떤 비용 항이 우리 과제에 가장 중요한가"를 데이터로 입증.

---

#### 대분류 3: 성능 평가 체계

**왜 필요한가:** 개선 효과를 정량적으로 입증하고, 향후 5세대 도입 여부를 판단하기 위한 근거를 확보한다. 평가 없이는 "3세대가 정말 나은지" 판단할 수 없다.

| 소분류 | 개발 항목 | 목적 |
|---|---|---|
| 3-1 | 리타게팅 성능 평가 프레임워크 | 메트릭 자동 측정 + 결과 로깅 시스템 |
| 3-2 | 평가 과제 셋 정의 | 난이도별 5종 과제(power grasp ~ 동적 유지) 표준화 |
| 3-3 | 1세대 기준선 평가 결과 | 기준선 정량 데이터 확보 (`models/direct_mapping`) |
| 3-4 | 1세대 vs 3세대 성능 비교 보고서 | 3세대 도입 효과 정량 입증 |
| 3-5 | 비용 함수 ablation 실험 보고서 | 비용 항별 기여도 분석 |
| 3-6 | 최종 3종 비교 성능 평가 보고서 | 1세대 vs 3세대 vs 3세대 최적화 전체 비교 |
| 3-7 | 향후 5세대 도입 필요성 판단 근거 | 3세대로 부족한 과제 식별 → 5세대 도입 여부 결정 |

→ **이것을 개발하는 이유:** 개발 효과를 객관적 수치로 보여주고, 향후 개발 방향 의사결정의 근거를 확보.

## 4. 개발 항목 간 의존 관계

```
[대분류 1] 입력 변환 시스템 ──────────────────────────────┐
  1-1 포맷 분석서                                         │
  1-2 매핑 테이블        ──→ 1-4 변환 노드 ───────────────┤
  1-3 좌표계 변환                                         │
                                                          ↓
[대분류 2-A] 1세대 기준선         [대분류 2-B] 3세대 AnyTeleop
  2-A-1 joint metadata ──→ 2-A-2  2-B-1 키포인트 정의 ──→ 2-B-2 비용 함수
          ↓                              ↓
      Direct Mapping 동작            2-B-3 솔버 ──→ 2-B-4 노드
          ↓                              ↓
[대분류 3] 평가                   3세대 리타게팅 동작
  3-1 평가 프레임워크                    ↓
  3-2 과제 셋 정의              [대분류 2-C] 비용 함수 최적화
      ↓                          2-C-1 ablation
  3-3 1세대 평가                  2-C-2 최적 가중치 ──→ 2-C-3 프리셋
      ↓                              ↓
  3-4 1세대 vs 3세대 비교        3세대 최적화 리타게팅 동작
      ↓                              ↓
  3-5 ablation 보고서            3-6 최종 3종 비교 보고서
                                      ↓
                                 3-7 향후 방향 판단
```

## 5. 정리: 한 문장 요약

> 원격조작의 파지 성공률을 SW 개선만으로 향상시키기 위해, 현재 주류인 3세대 리타게팅(AnyTeleop)을 Manus Glove + DG-5F 환경에 구현하고, Retargeting Objective 분석으로 과제별 최적화를 수행하며, 1세대 Direct Mapping 기준선과의 정량 비교로 개선 효과를 입증한다.
